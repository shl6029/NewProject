git 스터디

  • git : 버전관리 시스템
  • 분산버전관리 시스템 :
  • git config :
  • git init : 프로젝트에 깃저장소 생성

git의 추적(track)과 저장소에 저장(commit)

  • Working Directory : 작업하는 폴더
    • 작업이력이 저장되는 곳
    • 편집기(vs code)를 이용해 수정하면 바로 반영
    • untackted : 작업하는 파일인데 아무것도 하지 않았을 때
  • Staging Area : 작업하는 내역을 추적해서 저장할 준비하는 곳
    • 작업이력을 저장소에 저장하기 전에 추적하는 곳
    • 작업이력이 저장소에 어떻게 저장될 것인지 판단하게된다. (new file,modified,delete)
    • index :(index modified) 파일을 추적하고 저장할 준비가 된 상태 (git add file명 : 그 파일을 추적하겠다.)
    • git add . : 작업폴더 하위의 모든 파일을 추적하겠다.
  • local repository(.git) : git의 저장소(스냅샷이 저장되는 곳)
    • Staging Area(index) 에서 추적한 파일의 스냅샷을 저장하는 실질적인 저장소
    • git commit -m "코멘트"
    • git commit -a -m "코멘트" : git add 자동으로 해준다.
